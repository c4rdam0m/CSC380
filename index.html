CPU

In my CSC 270 class, I am learning how to create a CPU in an application that simulates how real circuits act. So far, I've created an arithmetic logic unit which can do simple math on 4 bit numbers. These numbers are stored in registers which connect and can be operated on by the ALU. Recently, we added a control logic unit, which stores the outcome of these operations into RAM. This is also controlled by a program counter, which is a major aspect of the CPU. Below is a screen shot of the complete CPU circuit I have developed so far.

The instruction word is created from 7 different qualities; Register or Immediate (constant int), Save Word, Load Word, Operation, Destination, Source 1, and Source 2. The destination and sources are all based on register addresses, of which there are 8 total. At first, the instruction word was just one input that had to be changed manually. But now, multiple instructions can be put into the memory, which is incremented with each tick from the clock by the PC. So an entire program can be run based on the instruction codes in memory, and the output will be put into RAM. For example, our assigment was to create a set of instructions that would reverse an array.
